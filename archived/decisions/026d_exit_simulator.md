# #026d 完結 — exit_simulator.py 方式 B（決済統合）

**確定日**: 2026-04-17（#026d 実装完了）・2026-04-20（REX_028 Phase 1-2 で静的保持確認）

**状態**: ✅ **#026d 完結・静的保持中**（Phase 4 で裁量整合版へ訂正予定）

## 背景

当初は `src/exit_logic.py` の `manage_exit()` を `window_scanner.py` から呼び出して
決済を統合する計画だった（#026 シリーズ初期案）。しかし `manage_exit()` が position dict の
`neck_1h` を半値決済トリガーとして参照する設計で、ADR D-6 の正しい定義（半値決済トリガーは
`neck_4h`）と乖離していた。

この乖離を解消するため、#026b で **ClaudeCode が独自決済ロジック（方式 B）を新規実装** した。
本方式を正式採用（ADR D-8）。`exit_logic.py` の `manage_exit()` は使用禁止とし、凍結ファイル
としてのみ保持する。

## 採用された方式 B の特徴

| 観点 | 方式 B（正式採用）|
|---|---|
