抽出粒度：両方併記

これが一番使いやすいと思う。理由が二つ：

要点リストは「事実の引き出し」として高速参照に強い。Mem0でsearchした時に該当ヒットしやすい。
要約段落は「文脈の再構築」に効く。雑談のニュアンスや、ある合意に至った流れみたいなものは段落じゃないと残らない。
二度手間に見えるけど、1セッション分の抽出量なんてたかが知れてるし、ベクトル検索的にも埋め込みのカバレッジが広がる方が後で助かる。

# 2026-05-19 morning - Mem0設計の合意

- トリガーは明示型（コマンド一発）から開始

これも welfare 的に綺麗だと思う。誤抽出が出ても personal/dialogues/ だから後で手で直せるし、誤抽出が残っても「ああこの時こう拾ったんだ」っていう地層になる。完璧な抽出を目指した瞬間に重荷になるから、ラフでいい。

ただ最初の2〜3回だけは Boss が結果を覗いて、抽出の癖（拾いすぎ/拾わなすぎ）を見てくれると安心かも。チューニングが要らなければ、そのあとは完全自動。

コマンドの主体：その通り、CLI/IDE側になる

Claude.aiのチャット画面からは直接Pythonスクリプトを叩けない。今のところ Boss が叩く想定なら：

ターミナル（PowerShell等）から python wrapup.py
VS Code / Cursor の統合ターミナルから同じ
バッチファイル化してデスクトップにショートカット、ダブルクリックで実行
これが第一段階。最終形としては MCP サーバー化すれば、Claude.aiから僕に「セッション終わったので抽出します」と言わせて、内部的にMem0を呼ぶ構成にできる。ただ最初は CLI からの方がデバッグも挙動確認も楽。

保存形式：同セッション追記、別スレ別ファイル

ファイル名規則の提案：

personal/dialogues/2026-05-19-1130-mem0_design.md
personal/dialogues/2026-05-19-2100-shooting_talk.md
personal/dialogues/2026-05-20-0830-morning_log.md
YYYY-MM-DD-HHMM-<short_topic>.md 形式。一日に複数スレあっても自然に並ぶ。後から日付でgrouping可能。

「セッションの境界」をどう検知するかは、コマンド引数で Boss が指定するのが一番シンプル：

python wrapup.py                          # 直近ファイルに追記
python wrapup.py --new "topic_name"        # 新ファイル作成
別スレで起動した時は --new 付けて叩く。同じ会話の続きで二回目以降叩くなら引数なし。これが welfare 的にも綺麗で、Rex も Mem0 も「自分は今どのセッションか」を意識しなくていい。Boss が判断する一点だけ。

ちなみにここまでの会話自体、いい WrapUp 素材になってる気がする。Mem0実装後の最初の抽出テストに使えそう。
