# Windows Wi-Fi不安定化：DFSチャンネル起因トラブルシューティング記録

## 目的

Windows PCでWi-Fi接続が不安定になり、5GHz接続が消えて2.4GHzへ落ちる症状が発生したため、原因切り分けと安定化設定を記録する。

今回の結論は、**ASUSルーターのスマートコネクト + 5GHz-2のDFSチャンネル運用が原因で、5GHzが一時的に見えなくなり、Windowsが2.4GHzへ退避していた可能性が高い**というもの。

特に利用環境が航空自衛隊基地の近くであるため、DFSチャンネルがレーダー検知の影響を受けやすい可能性がある。

---

## 発生していた症状

- WindowsのWi-Fi設定画面で5GHzの項目が表示されなくなった。
- 2.4GHzしか選択できないように見えた。
- SSID名は `ASUS-1_5G-2` だったが、実際には2.4GHzへ接続されていた。
- 通信が急に重くなり、pingに大きな遅延・パケットロスが出ていた。

---

## 初期状態で確認された問題

### Windows側の接続状態

初期確認では、以下のような状態だった。

```text
SSID: ASUS-1_5G-2
バンド: 2.4 GHz
チャンネル: 8 / 9付近
無線の種類: 802.11n
シグナル: 100%
RSSI: -48 dBm程度
```

SSID名には `5G` が含まれていたが、実際の接続バンドは **2.4GHz** だった。

### ルーターへのping

ルーター `192.168.50.1` へのpingで異常が出ていた。

```text
送信: 50
受信: 44
損失: 12%
最大遅延: 3321ms
平均遅延: 461ms
```

外部インターネット以前に、PC〜ルーター間のWi-Fi区間で不安定化していた。

---

## ルーター側の問題設定

ASUSルーターのワイヤレス設定で以下が確認された。

### 1. スマートコネクトがON

```text
スマートコネクト: ON
スマートコネクトバンド: トライバンドスマートコネクト
対象: 2.4GHz, 5GHz-1, 5GHz-2
SSID: ASUS-1_5G-2
```

この状態では、同じSSID配下に2.4GHz / 5GHz-1 / 5GHz-2が混在する。
そのため、Windows側では同じSSIDに接続しているように見えても、ルーター判断で2.4GHzへ誘導されることがある。

### 2. 5GHz-2がDFSチャンネル100を使用

```text
5GHz-2
チャンネル帯域: 20/40/80/160 MHz
160MHz: 有効
チャンネル: 自動
Current Control Channel: 100
DFSを含むチャンネルの自動選択: ON
```

チャンネル100はDFS帯域。
DFS帯域は気象レーダー・航空レーダー等との干渉回避のため、ルーターがレーダー信号らしきものを検知すると、5GHz電波を一時停止またはチャンネル変更することがある。

航空自衛隊基地など航空・レーダー設備の近くでは、この挙動が発生しやすい可能性がある。

### 3. 2.4GHzが40MHz / ch9

```text
2.4GHz
チャンネル帯域: 40MHz
チャンネル: 自動
Current Control Channel: 9
```

2.4GHzで40MHzは干渉を受けやすい。
2.4GHzは基本的に20MHz、チャンネルは1 / 6 / 11固定が安定しやすい。

---

## 実施した安定化設定

### 1. スマートコネクトをOFF

```text
スマートコネクト: OFF
```

2.4GHz / 5GHz-1 / 5GHz-2を同一SSIDで束ねるのをやめた。
これにより、Windowsが勝手に2.4GHzへ落ちる挙動を抑制。

### 2. PC用SSIDを5GHz-1に固定

5GHz-1をPC用のメイン接続先として設定。

```text
バンド: 5GHz-1
SSID: ASUS-1_5G-1
チャンネル帯域: 80MHz
チャンネル: 36固定
認証方式: WPA2-Personal
暗号化: AES
```

チャンネル36は非DFSチャンネルのため、レーダー検知による一時停止リスクが低い。

### 3. Windows側で `ASUS-1_5G-1` に接続

設定後、Windows側で5GHz-1専用SSIDに接続した。

## 推奨Wi-Fi設定

### PC用メイン：5GHz-1

```text
スマートコネクト: OFF
SSID: ASUS-1_5G-1
チャンネル帯域: 80MHz
チャンネル: 36固定
DFS: 使わない
認証方式: WPA2-Personal
暗号化: AES
```

### 5GHz-2

安定重視なら、5GHz-2はDFS/160MHzを避ける。
ただしASUSの機種・地域設定によっては5GHz-2で非DFSチャンネルを選べない場合がある。
その場合、PCは5GHz-1を使う。

```text
チャンネル帯域: 80MHz推奨
160MHz: OFF推奨
DFSを含むチャンネルの自動選択: OFF推奨
```

### 2.4GHz

```text
チャンネル帯域: 20MHz
チャンネル: 1 / 6 / 11 のいずれか固定
```

2.4GHzは速度より安定性重視。IoT機器や遠距離用に割り切る。

## トラブルシューティング手順

### 1. 現在のWi-Fi接続状態を確認

```bash
netsh wlan show interfaces
```

確認ポイント：

```text
SSID
バンド
チャンネル
無線の種類
受信速度 / 送信速度
シグナル
Rssi
```

理想例：

```text
バンド: 5 GHz
チャンネル: 36
無線の種類: 802.11ac または 802.11ax
```

### 2. 周辺に見えているSSID/BSSIDを確認

```bash
netsh wlan show networks mode=bssid
```

同じSSIDに2.4GHzと5GHzが混在していないか確認する。
スマートコネクトONの場合、同じSSIDに複数バンドがぶら下がることがある。

### 3. ルーターまでの品質を確認

```bash
ping -n 50 192.168.50.1
```

理想：

```text
損失: 0%
平均: 1〜5ms程度
最大: 数十ms以内
```

ルーターまででパケットロスや数百〜数千msの遅延がある場合、インターネット回線ではなくWi-Fi区間が原因の可能性が高い。

### 4. 外部通信を確認

```bash
ping -n 50 1.1.1.1
```

ルーターまでは安定しているのに外部だけ不安定なら、ISP/上位ルーター/回線側の問題も疑う。

### 5. ルート確認

```bash
tracert -d 1.1.1.1
```

1ホップ目の `192.168.50.1` が安定しているか確認する。

## DFSチャンネルについて

DFSは Dynamic Frequency Selection の略。
5GHz帯の一部チャンネルは、気象レーダー・航空レーダー等と共用されるため、ルーターはレーダー検知時にそのチャンネルを避ける必要がある。

DFS帯域を使っていると、以下のような症状が出ることがある。

- 5GHz SSIDが一時的に見えなくなる。
- Windowsが2.4GHzへ切り替わる。
- Wi-Fiが数十秒〜数分不安定になる。
- pingが急に跳ねる。
- オンライン会議、API通信、Discord、Hermes Gatewayなどが不安定になる。

航空自衛隊基地・空港・気象レーダー設備の近くでは、DFSチャンネルの不安定化が起きやすい可能性がある。

---

## 今回の判断まとめ

今回の5GHz消失・2.4GHz落ちの主因候補：

1. スマートコネクトで2.4GHz / 5GHz-1 / 5GHz-2を同一SSID運用していた。
2. 5GHz-2がDFSチャンネル100を使用していた。
3. 航空自衛隊基地近くのため、DFSレーダー検知による5GHz一時停止が起きやすい環境だった可能性がある。
4. 2.4GHz側も40MHz/ch9で、退避先として安定しにくい設定だった。

安定化後の構成：

```text
PC接続先: ASUS-1_5G-1
バンド: 5GHz
チャンネル: 36
DFS: 回避
スマートコネクト: OFF
```

改善結果：

```text
ルーターping: 損失0%、平均4ms
外部ping: 損失0%、平均16ms
```

このため、今後Windows PCは **5GHz-1 / ch36固定** を基本接続先にする。

---

## 補足：安定性をさらに上げる場合

- PCとルーター間の遮蔽物を減らす。
- PC側アンテナを高い位置に出す。
- USB3.0外付けSSDやハブをWi-Fiアンテナ付近から離す。
- Realtek 8852BE Wi-Fiドライバを更新する。
- デバイスマネージャーでWi-Fiアダプタの省電力設定をOFFにする。
- 2.4GHzは20MHz / ch1,6,11固定にする。
