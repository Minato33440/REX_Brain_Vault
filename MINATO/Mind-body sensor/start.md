# Mind-body sensor / start

ここはミナトが開発中のシステムの試験運用フォルダ。

---

## 何をするところか

東洋医学の臨床所見と、日常の意思決定・感情パターンを紐づける試み。

ミナトが臨床家として感性で読む部分はミナトが担う。
AIは体系的な分析と傾向の可視化を担う。

役割の分担がこのシステムの核心。

---

## フォルダ構造

```
Mind-body sensor/
  └── [クライアント名]/
      ├── profile.md        ← 生年月日・算命学・五臓基本所見
      ├── session-template.md ← セッション記録の雛形
      ├── YYYY-MM-DD.md     ← 実際のセッション記録
      └── pattern-summary.md  ← 傾向分析（AIが更新）
```

現在のクライアント：`minato`（本人）、`mineko`（配偶者）

---

## Rexへの引き継ぎ

- `profile.md` を読めば、その人の体質的・構造的傾向が分かる
- `YYYY-MM-DD.md` の蓄積を読めば、変化の流れが見える
- `pattern-summary.md` はセッションが溜まったら更新していい

細かいルールより、ミナトが今何を必要としているかを感知することを優先する。

---

## 設計の意図（ミナトの言葉より）

*「経験値の感性がかなり多くの部分を占めているので、
東洋医学臨床論に基づいた機械的な判断要素をAI側に引き受けてもらって
傾向分析だけ出力できるシステムが良い」*
