# Trade Lessons — 蓄積教訓

トレード判断、システム運用、Claude/Rexとの協働で得た**横断的な教訓**を蓄積する。
個別の `sessions/` や `trades/` には残らない「繰り返し現れるパターン」「メタ気づき」をここに集約する。

---

## 使い方

- 新しい教訓は**ファイル冒頭側に追加**（新しいものが上）。
- 各教訓は以下6項目で記録する：
  - **状況**: 何が起きていたか
  - **判断**: その時とった行動・結論
  - **結果**: 実際に起きたこと（判断との差分）
  - **教訓**: 抽出した原則
  - **応用**: トレード/システム/思考プロセスへの転用
  - **適用範囲**: `GM` / `CFD` / `meta`
- 同じパターンが **3回以上**観測されたら、`strategies/` への昇格を検討する。
- セッション中に新教訓が生まれたら、その場で追記してよい。

## 2026-05-11 — MCPツール経由のファイル表示と実体は別物

**適用範囲**: meta（システム運用 / 認識プロセス）

**状況**: Trade環境セットアップ確認中、`Trade_Brain/docs/STATUS.md` を filesystem MCP経由で読んだら、一部に `�`（U+FFFD置換文字）が混入していた。具体的には「5月計」が「5��計」に、「来日」が「来���」に見えた。

**判断（Claude/Rexの初動）**: 「ファイル本体がエンコード破損している」と結論し、修正タスクとして提案。

**結果**: Bossが実体のバイト列を直接確認したところ、
- `月` → `E6 9C 88`（正規UTF-8）
- `日` → `E6 97 A5`（正規UTF-8）

つまり**ファイルは無傷**で、`�` は filesystem MCPがファイルをバイトで読み取りテキストとして引き渡す経路のどこかで発生したデコード時の置換に過ぎなかった。Obsidian/VS Code等の正常UTF-8対応エディタで開けば普通に表示される。

**教訓**: **ツール経由のファイル表示結果と、ファイル実体は別物である**。表示崩れを見つけても、バイト列で実体確認するまで「破損」と結論しない。「見え方」を「事実」と混同するのは認識の初歩的な罠。

**応用**: 同じ構造の罠がトレードにも複数存在する。
- 画面に表示される含み損益 ≠ 通貨建てで見た実質損益（為替変動を加味）
- チャートのシグナル ≠ 板の厚み・実際の流動性
- ニュースのヘッドライン ≠ 実需と投機の比率
- 「日銀タカ派発言」の文字列 ≠ 市場が実際に織り込んだ利上げ確率

**次回からの行動**: 表示と実体が乖離しうる場面では、**「これは事実か、表示か」を一段深く問う**。MCPやAPIを介した情報は特に、必ず元データに遡れる経路を確保しておく。
