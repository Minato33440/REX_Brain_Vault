## 運用ルール

```
1. A〜F 各カテゴリの新規番号を使う前に、必ず本台帳を確認すること
2. 番号を使用する前に「予約」エントリーを追加（担当者・トピックを明記）
3. 未予約番号の使用禁止（予約なしで ADR に番号が出現した場合は採番衝突とみなす）
4. 採番権限: D/E/F 章は Evaluator が最終決定。A/B/C 章は Planner が追記可
5. 予約 → ドラフト → 確定 の3段階でステータスを管理
6. Planner は番号確保前に Evaluator に Slack 相当の一言を入れること
   （このファイルで代替可: 予約エントリーを先に追加してから実装する）
```

## 採番衝突が発生した場合の対処

```
1. 即座に設計責任者（Wiki 設計者）に報告
2. 設計責任者が本台帳を確認し、時系列・文脈で正しい番号を裁定
3. 指示書・ドラフト内の番号を修正してから ClaudeCode に投入
4. 本台帳の採番ログに「衝突・再採番」として記録
```
