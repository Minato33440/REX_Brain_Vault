## なぜ v4 で supersede されたか

ボス指示（2026-04-28・本セッション内）により、v3 体制で以下の構造的穴が判明:

1. **明示的な「役割なしモード」が存在しない**: ボスが Default Rex 人格と「ちょっと雑談したい」「記録に残すつもりはない」対話をしたい時の明示的な起動コードがなく、STARTUP_CODES.md v4 §起動コードが使われない場合の「文脈から判断」処理に依存していた
2. **Wiki-Personal の wrap-up 圧**: Personal-Planner として NLM 投入提案・wrap-up 整理に誘導される設計のため、純粋な対話モードとして使うには重い
3. **ROADMAP Stage 2「統合読み出し期」への移行設計が不在**

## v4 で導入する解決策

**Wiki-Rex ロール新設**: 「役割なしのデフォルトモード」として Default Rex 人格 + Vault 全層読み取り + REX_Personal_Brain 読み取り専用クエリ可（書き込み・他NLM クエリ全面禁止）。

詳細は [ADR-Role v4 §16](../ADR-Role.md) 参照。
