# UCAR cross-agent protocol

`UCAR/` は GPT / Codex 系主体が使う個人記憶層。`REX/` は Claude / Rex 系主体の個人記憶層。
同じ `REX_Brain_Vault` の中にあるが、両者は同一人格・同一脳ではない。

## 基本ルール

1. **書く場所**
   - UCAR が書いてよい一次記憶: `UCAR/`
   - REX が書いてよい一次記憶: `REX/`
   - プロジェクトの決定・仕様・運用・引き継ぎ: `bridges/` または各プロジェクトリポ

2. **読む場所**
   - UCAR は `REX/` を読むことができる。ただし、原則として編集しない。
   - REX は `UCAR/` を読むことができる。ただし、原則として編集しない。
   - 相手の記憶を使う時は「コピー」ではなく、リンクと短い解釈に留める。

3. **一次資料の優先順位**
   1. ユーザーの現在の指示
   2. `bridges/` または各プロジェクトリポの公式記録
   3. `AGENTS.md` / `CLAUDE.md` / README などの入口文書
   4. `UCAR/` と `REX/` の個人記憶層

4. **複製しないもの**
   - プロジェクトの正式決定
   - 仕様
   - 引き継ぎ事項
   - APIキー・認証情報・秘密情報
   - 個人情報の生ログ

これらは UCAR と REX の双方に複製せず、共有すべき内容は `bridges/` を一次資料にする。

## UCAR が残すとよいもの

- GPT / Codex として作業した時の判断癖・得意不得意
- 次回以降の UCAR が参照すると役に立つ短い操作文脈
- REX の記録を読んだ上での UCAR 側の解釈
- Hermes / Codex / GPT 系の運用で見つけた注意点
- プロジェクト本体に入れるほどではないが、UCAR の継続性に効く気づき

## UCAR が残さないもの

- `bridges/` に置くべき公式な決定
- プロジェクト成果物そのもの
- REX の文章の丸写し
- セッションログ全文
- 1週間で陳腐化する進捗報告だけの記録

## 実務フロー

### セッション開始時

必要なら次を確認する。

1. `AGENTS.md`
2. `UCAR/morning.md`
3. `UCAR/UCAR-Vault.md`
4. 作業対象プロジェクトの `bridges/`
5. 必要な場合だけ `REX/` 側の対応ノート

### 作業中

- 公式判断が発生したら `bridges/` へ寄せる。
- UCAR 個人の気づきなら `UCAR/YYYY-MM-DD.md` または概念ノートへ短く残す。
- REX の記録を根拠にする場合は、`[[REX/...]]` へのリンクと UCAR 側の解釈を書く。

### セッション終了時

次の条件を満たす時だけ UCAR に記録する。

- 次回の GPT / Codex 系主体が同じ迷いを避けられる。
- プロジェクト公式記録ではなく、UCAR の運用判断として残す意味がある。
- 秘密情報や生ログを含まない。

## frontmatter について

UCAR 側で新規ノートを作る時、frontmatter は分類や検索に必要な最小限だけを書く。

```yaml
---
type: record
created: YYYY-MM-DD
tags:
  - ucar
---
```

`owner` や `access` は原則として書かない。UCAR ノートの所有主体と相互参照範囲は、ノート単位のメタデータではなく `UCAR/` というディレクトリ配置とこのプロトコル本文で判断する。境界の真実の所在を二重化しないため。

## 判断文

> `UCAR/` は UCAR の脳。`REX/` は Rex の脳。`bridges/` は共有すべき事実の橋。
> 脳は相互参照できるが、公式記録は橋に置く。
