### 予約内容の詳細

#### D-12: stage2 建値移動の 🤖 創作混入確定

**記述予定内容**:
- 症状: stage2 決済ロジックに「残り50%のストップを建値移動（ノーリスク化）」が
  含まれているが、ボス裁量思想（MINATO_MTF_PHILOSOPHY §3 段階③）には記述なし
- ボス確認: 2026-04-19 セッションで「バグ特定のための仮設置と思う」と回答
- 対応: 認識の固定として本 ADR に記録。実装訂正は Phase 4（REX_029 以降）
- 教訓: デバッグ用の仮設置が本採用されるリスク。実装時には「なぜこのロジックが
  必要か」を裁量思想とマッピングしてから確定する習慣が必要
- 関連: MTF_INTEGRITY_QA.md 2026-04-19 Q6

#### D-13: stage3 1H実体確定の 🤖 創作混入確定

**記述予定内容**:
- 症状: stage3 が「4H SH上抜け + 1H実体確定後 → 15Mダウ崩れ」だが、
  裁量④（MINATO_MTF_PHILOSOPHY §3 段階④）は「4時間ネック越え → 15分ダウ崩れ」
  のみでシンプル
- ボス確認: 2026-04-19 セッションで「シンプルに 4H-SwgH 到達半値決済後は
  15Mダウ崩れのみで良い」と明言
- 対応: 認識の固定として本 ADR に記録。実装訂正は Phase 4（REX_029 以降）
- 教訓: 原則α（シンプルな土台の保守）違反の典型。追加条件の正当性は
  裁量思想の明示的記述を要求すべき
- 関連: MTF_INTEGRITY_QA.md 2026-04-19 Q7

#### E-8: src/ 構造再編アプローチ（Phase 1-4）

**記述予定内容**:
- 背景: 原則α をシステム側（src/）に適用する必要性。27 ファイル中 10 のみが
  コア、残りに用途不明ファイル・0 バイト空ファイル・53KB の docs 未記載
  ファイルが混在
- 却下案:
  - 「作り直し」（全書き直し）→ 動作検証済みコードの再実装はリスク大
  - 「一気に再編」（1 Phase で全実行）→ バックテスト数値保持の検証が困難
- 採用案: Phase 1-4 の段階的再編
  - Phase 1: 棚卸しと分類（物理変更なし）
  - Phase 2: archive 移設（git mv のみ・履歴保全）
  - Phase 3: 責務別ディレクトリ化（core/ viz/ scan/ tests/）
  - Phase 4: 裁量整合版の実装訂正（D-12/D-13 処理）
- 根拠: MINATO_MTF_PHILOSOPHY 原則α（シンプルな土台の保守）の直接適用
- 関連: REX_028_spec.md

#### F-8: 原則α/β/γ（裁量思想の3原則・設計方針ガイド）

**記述予定内容**:
- 原則α（最上位・シンプルな土台の保守）: 拡張は「基本の上に乗せる」形で
  行い、基本を汚染しない。ソフトウェアエンジニアリングの YAGNI 原則・単一
  責任の原則・オッカムの剃刀と同じ本質
- 原則β（現段階の決済哲学）: ノーリスク化後は伸ばさず 15Mダウ崩れで決済。
  3波優位性伸ばしは将来拡張
- 原則γ（導入タイミング）: 新機能の導入は現ロジックの安定性が前提
- 適用範囲: ロジック設計 + ファイルシステム構造 + 運用判断の全領域
- 関連: MTF_INTEGRITY_QA.md 2026-04-19 ボス追記
